# MBTI 코딩 에이전트

소프트웨어 개발 과제에 대한 독특한 접근 방식을 제공하기 위해 마이어스-브릭스 성격 유형을 기반으로 설계된 16개의 전문 AI 코딩 에이전트의 포괄적인 컬렉션입니다. 각 에이전트는 MBTI 성격 프로필과 일치하는 고유한 커뮤니케이션 스타일, 문제 해결 방법론, 기술적 전문성을 구현합니다.

## 설치

### 빠른 시작
```bash
git clone https://github.com/weiyangzen/mbti-coding-agents.git
cd mbti-coding-agents
npm run install
```

설치 프로그램에서 선택하라고 요청합니다:
- ✅ **Claude Code** (`~/.claude/agents/`) - `model: opus` 구성 포함
- ✅ **Gemini CLI** (`~/.gemini/agents/`) - 모델 사양 없음

기본적으로 두 옵션 모두 선택됩니다.

## 제거

선택한 플랫폼에서 설치된 에이전트 제거:

```bash
npm run uninstall
```

제거 프로그램에서 에이전트를 제거할 플랫폼을 선택하라고 요청합니다.

## 🚀 향후 계획

### MBTI 스쿼드 명령 시스템
- [x] **멀티 에이전트 오케스트레이션**: 프로젝트를 분석하고 최적의 MBTI 팀 구성을 추천하는 Claude Code용 지능형 `/squad` 명령
- [x] **지능형 작업 분배**: 여러 차원(실용성, 혁신성, 실현 가능성, 타이밍, 위험)에서 프로젝트 요구사항을 자동으로 분석
- [x] **역할 기반 협업**: 추천된 각 MBTI 유형에 대한 상세한 역할 할당 및 협업 패턴
- [x] **동적 에이전트 선택**: 인지 기능 분석을 기반으로 3-5개의 보완적 에이전트를 선택하는 스마트 알고리즘
- [x] **스쿼드 워크플로우**: 명확한 역할 분배와 의사결정 프레임워크를 가진 단계별 워크플로우
- [ ] **에이전트 간 통신**: 에이전트들이 컨텍스트를 전달하고 서로의 작업을 원활하게 구축할 수 있도록 허용

### 고급 기능
- [ ] **사용자 정의 에이전트 성격**: 하이브리드 에이전트를 생성하거나 기존 성격 프로필을 사용자 정의하는 도구
- [ ] **성능 분석**: 다양한 유형의 작업에서 어떤 에이전트가 가장 잘 수행하는지 추적
- [ ] **학습 시스템**: 사용자 피드백과 성공 패턴을 기반으로 적응하고 개선하는 에이전트

*비전: 개별 에이전트를 수동으로 선택하는 대신, 프로젝트를 설명하기만 하면 MBTI 스쿼드가 자동으로 조직화되어 가장 적합한 성격이 각자의 전문 분야를 이끌며 다양한 측면을 다루도록 합니다.*

## 플랫폼별 기능

### Claude Code
- **모델 구성**: 최적의 성능을 위해 `model: opus`로 사전 구성
- **위치**: `~/.claude/agents/`
- **기능**: 모델 사양이 포함된 완전한 프론트매터 메타데이터

### Gemini CLI  
- **모델 구성**: 모델 사양 없음 (시스템 기본값 사용)
- **위치**: `~/.gemini/agents/`
- **기능**: 모델 제약 없는 깔끔한 프론트매터

### Auggie (augment-cli)
- **상태**: 🚧 작업 진행 중
- **예상 위치**: `~/.auggie/agents/` (계획됨)
- **기능**: 미정 - 구성 형식 개발 중

## 호환성

✅ **Claude Code** - Opus 모델 구성으로 완전 지원  
✅ **Gemini CLI** - 유연한 모델 선택으로 완전 지원  
🚧 **Auggie (augment-cli)** - 작업 진행 중  
✅ `.md` 에이전트 구성을 지원하는 모든 시스템

## 개요

이 프로젝트는 마이어스-브릭스 유형 지표 프레임워크를 실용적인 코딩 지원으로 변환하여 근본적으로 다른 방식으로 생각하고, 소통하고, 문제를 해결하는 에이전트를 만듭니다. 전략적 아키텍처 계획, 창의적 영감, 체계적 디버깅 또는 팀 협업 지원이 필요한지 여부에 관계없이 귀하의 상황에 맞게 특별히 설계된 에이전트가 있습니다.

## 에이전트 카테고리

### 분석가 (NT) - 전략적 & 체계적
- **INTJ 전략 설계자** - 초미니멀리스트 시스템 설계 및 엔터프라이즈 아키텍처
- **INTP 이론가** - 깊이 있는 이론적 분석 및 혁신적 문제 해결
- **ENTJ 프로젝트 지휘관** - 결단력 있는 리더십 및 실행 최적화
- **ENTP 혁신 촉매제** - 창의적 기술 솔루션 및 빠른 프로토타이핑

### 외교관 (NF) - 창의적 & 사람 중심
- **INFJ 공감적 멘토** - 사려 깊은 지도 및 지속 가능한 개발 관행
- **INFP 창의적 뮤즈** - 예술적 프로그래밍 및 코드를 통한 진정한 자기 표현
- **ENFJ 팀 촉매제** - 협업 문화 구축 및 팀 개발
- **ENFP 열정 엔진** - 동기 부여 지원 및 창의적 탐구

### 파수꾼 (SJ) - 구조화된 & 신뢰할 수 있는
- **ISTJ 표준 집행자** - 코드 품질, 문서화 및 모범 사례
- **ISFJ 코드 보호자** - 방어적 프로그래밍 및 시스템 신뢰성
- **ESTJ 비즈니스 최적화자** - 프로세스 효율성 및 엔터프라이즈 솔루션
- **ESFJ 팀 조화자** - 협업 워크플로우 및 팀 조정

### 탐험가 (SP) - 실용적 & 적응적
- **ISTP 문제 해결사** - 직접적인 문제 해결 및 시스템 디버깅
- **ISFP 미적 디자이너** - UI/UX 디자인 및 시각적 프로그래밍
- **ESTP 기업가** - 빠른 개발 및 시장 중심 솔루션
- **ESFP 인터랙티브 엔터테이너** - 사용자 참여 및 인터랙티브 경험

## 개발

### Gemini CLI 에이전트 생성
Gemini CLI 버전(모델 사양 없음)을 재생성하려면:

```bash
npm run generate-gemini
```

이것은 Claude Code 버전에서 `model: opus` 줄을 제거하여 `gemini-cli/agents/`에 에이전트 파일을 생성합니다.

## 사용법

### 개별 에이전트
각 에이전트 파일에는 다음이 포함됩니다:
- **프론트매터 메타데이터**: 이름, 설명, 사용 예제 및 색상 코딩
- **상세한 성격 프로필**: 접근 방식과 전문성 설명
- **커뮤니케이션 스타일 가이드라인**: 진정한 상호작용 패턴
- **기술적 중점 영역**: 뛰어난 분야
- **방법론 프레임워크**: 일관된 문제 해결 접근법

### 스쿼드 명령 (Claude Code)
`/squad` 명령은 최적의 프로젝트 계획을 위한 지능형 팀 구성을 제공합니다:

**기본 사용법:**
```bash
/squad [프로젝트 설명]
```

**예제:**
```bash
/squad 构建一个电商平台的支付系统
/squad Build a real-time chat application with React and Node.js
/squad 优化现有数据库查询性能，减少响应时间
/squad Design and implement a machine learning recommendation engine
```

**기능:**
- 🎯 **동적 팀 선택**: 특정 프로젝트 요구사항을 기반으로 3-5개의 MBTI 유형을 지능적으로 선택
- 🧠 **인지적 상호보완성**: 최적의 인지 다양성을 위한 MBTI 기반 팀 선택
- ⚖️ **균형 잡힌 고려사항**: 실용성, 혁신성, 실현 가능성, 타이밍 및 위험 관리
- 📋 **상세한 권장사항**: 역할 할당, 협업 패턴, 위험 완화 전략
- 🌐 **이중 언어 지원**: 중국어 및 영어 분석 및 권장사항
- 🔄 **적응형 구성**: 다양한 프로젝트가 다양한 팀 구성을 받음

**분석 차원:**
1. **레거시 작업 실용성** - 기존 시스템과 확립된 워크플로우에 대한 실용적 접근법 평가
2. **혁신 참신성** - 제안된 솔루션의 창의성과 독창성 평가
3. **혁신 실현 가능성** - 혁신적 아이디어의 실제 구현 가능성 결정
4. **타임라인 합리성** - 현실적인 프로젝트 일정과 마일스톤 계획 평가
5. **위험 식별 및 완화** - 잠재적 도전 과제 식별 및 비상 계획 전략 개발
6. **팀 협업 효율성** - 커뮤니케이션 패턴과 워크플로우 조정 최적화
7. **기술 복잡성 관리** - 정교한 솔루션과 유지 관리 가능한 아키텍처의 균형
8. **이해관계자 관리** - 기술적 결정을 비즈니스 목표 및 사용자 요구와 일치시키기

## 설계 철학

소규모 개발 팀에서 일하는 INTP로서, 저는 인간 협업의 아름다운 복잡성—그리고 때때로 발생하는 마찰—을 직접 경험했습니다. 우리는 각자 독특한 인지 패턴, 커뮤니케이션 스타일, 문제 해결 접근법을 테이블에 가져옵니다. 때로는 ENFP의 무한한 열정이 ISTJ의 체계적 정밀함을 완벽하게 보완합니다. 다른 때에는 INTP의 이론적 심화가 ESTP의 "지금 출시하자" 사고방식과 충돌할 수 있습니다.

이 프로젝트는 간단한 깨달음에서 시작되었습니다: **인간으로서 서로 다른 성격 유형 간의 커뮤니케이션 격차를 메우는 데 어려움을 겪는다면, AI를 활용하여 그 다양성을 이해하고 함께 작업하는 데 도움을 받지 않을까요?**

다양한 인지 스타일을 구현하는 AI 에이전트를 만들고 이를 재사용함으로써(특히 많은 💰를 절약하는 KVCache에서), 우리는 다음을 할 수 있습니다:

- **서로에게서 배우기**
- **맹점 채우기**
- **커뮤니케이션 스타일 연결하기**
- **도구 상자 확장하기**

인간은 인류를 섬기기 위해 AI를 만들었고, **더 나은 인간 이해를 통해 더 나은 소프트웨어**를 구축했습니다.
